### Frappe n8n

**Version:** 16.0.1

Frappe n8n is a powerful provider plugin for the Frappe Playbook ecosystem. It seamlessly integrates your Frappe applications with n8n, the leading fair-code workflow automation tool. By acting as a bridge between Frappe Playbook and n8n, this app allows you to trigger, manage, and monitor complex n8n workflows directly from within your Frappe environment.

**Key Features:**
- **Automated Workflow Execution:** Trigger n8n workflows automatically based on Frappe document events (e.g., when a Lead is created or a Deal is won).
- **Playbook Integration:** Fully compatible with the Frappe Playbook architecture, allowing you to define conditions and filters for when workflows should run.
- **Execution Tracking:** Monitor the status of your n8n workflow executions (Running, Success, Error, Canceled) directly from the Frappe interface.
- **Seamless Authentication:** Manage n8n API credentials and webhook configurations securely within Frappe.

### Triggers, Webhooks & Callback Schema

By default, Frappe generates an empty workflow for you in n8n. You are free to design your workflow with **any trigger** you like (Cron, RSS, Polling, etc.).

If you want Frappe to trigger the workflow automatically on Frappe document events, you must manually add a **Webhook trigger node** in the n8n UI. Frappe will automatically sync your changes and detect the webhook URL the next time you save the Playbook in Frappe (or within a few minutes via the background scheduler).

#### Outbound Webhook (Frappe → n8n)

*This only applies if your workflow uses a Webhook node to receive events from Frappe.*

**HTTP Headers:**
- `Content-Type`: `application/json`
- `Frappe-Playbook-Execution-Name`: `<execution_name>` (Used for identity and idempotency)
- `Authorization`: `Bearer <webhook_security>` *(If configured in n8n Settings)*

**JSON Payload:**
```json
{
  "name": "TASK-001",
  "doctype": "ToDo",
  "description": "...",
  "...": "..."
}
```

#### Inbound Callback (n8n → Frappe)

To log the execution cycle inside Frappe, your n8n workflow should include an HTTP Request node at the end to call back to Frappe.

**Endpoint:** `POST <your_frappe_url>/api/method/frappe_n8n.playbook_execution.callback`

The callback endpoint features a **dynamic schema and upsert behavior**. It will attempt to update any field passed in the payload that actually exists on the `Playbook Execution` DocType (including any custom fields you've added).

**Mandatory Fields for Updating an Existing Execution (Frappe-Triggered):**
- `execution_name`: Must perfectly match the `Frappe-Playbook-Execution-Name` header from the trigger webhook.

**Mandatory Fields for Creating a New Execution (n8n-Triggered, e.g. Cron/RSS):**
If the workflow was triggered by n8n (so no `execution_name` was sent from Frappe), the callback will dynamically create a new `Playbook Execution` document for you.
- `playbook`: The exact name of the Playbook document in Frappe.

**Allowed/Recommended Fields:**
- `status`: Update the status of the execution (e.g., `success`, `error`, `canceled`, `waiting`).
- `execution_data`: Any arbitrary JSON object representing the workflow result, logs, or payload generated by the n8n process.
- `n8n_execution_id`: The ID of the n8n execution, highly recommended to enable the "Debug in Editor" feature inside the Frappe UI.
- `*`: Any other valid field existing on the `Playbook Execution` document schema.

**Disallowed Fields:**
For system integrity, attempts to update structural and core metadata fields (such as `name`, `owner`, `creation`, `playbook`, `reference_doctype`, `reference_name`, etc.) will be securely and silently ignored.

**Example JSON Payload (Updating a Frappe-triggered execution):**
```json
{
  "execution_name": "{{ $request.header['frappe-playbook-execution-name'] }}",
  "status": "success",
  "execution_data": {
    "output_value": "{{ $json.processed_data }}",
    "logs": "Workflow completed smoothly."
  },
  "n8n_execution_id": "{{ $execution.id }}"
}
```

**Example JSON Payload (Logging an n8n-triggered cron execution):**
```json
{
  "playbook": "Daily Customer Sync",
  "status": "success",
  "execution_data": {
    "records_synced": 50
  },
  "n8n_execution_id": "{{ $execution.id }}"
}
```
*Note: The callback endpoint will respond with the fully created or updated `Playbook Execution` document in its response body.*

### Installation

You can install this app using the [bench](https://github.com/frappe/bench) CLI:

```bash
cd $PATH_TO_YOUR_BENCH
bench get-app $URL_OF_THIS_REPO --branch main
bench install-app frappe_n8n
```
